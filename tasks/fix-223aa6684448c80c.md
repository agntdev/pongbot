# fix-223aa6684448c80c — /count command crashes with ReferenceError — uses undefined pingStore and PING_COUNT_KEY

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 PONG

In `src/bot.ts:38-42`, the `/count` command handler references `pingStore.read(PING_COUNT_KEY)`, but neither `pingStore` nor `PING_COUNT_KEY` is imported or defined anywhere in the file or codebase. This will throw a `ReferenceError` at runtime, making the `/count` command completely broken.

The callback query handler for `cmd:count` (line 57-61) correctly uses `getCount()` from `./persistence.js` (imported on line 2). The `/count` command handler should do the same:

```typescript
bot.command("count", async (ctx) => {
  const count = await getCount();
  await ctx.reply(`Total pings served: ${count}`);
});
```

This bug means the E1T2 spec (`tests/specs/E1T2.json`) that tests `/count` will fail — the bot will crash instead of replying.
